Architecture Overview

A 1-layer causal decoder with d_model=5, qk_dim=4, split into a 2D token subspace and a 3D position subspace. Sinusoidal positions (frozen) and every component tightly optimized for parameter efficiency.

Token Embeddings

All 10 digits live on a circular arc of radius 6.06. Only 3 parameters define the entire embedding table. The y-coordinate increases monotonically with digit value, which is critical for the model's carry detection.

Circular Arc

Hover over any digit to see its exact coordinates.

Y-Coordinate vs Digit Value

The near-linear y-coordinate makes digit sum encodeable as an average.

Key insight: When the model averages two digit embeddings (via 50/50 attention), the y-coordinate of the average encodes x+y. The FFN then detects whether x+y ≥ 10 to determine carries.

Position Encoding

Positions are 3D vectors. The 10 digit positions use frozen sinusoidal encoding (shared across X, Y, Z). The carry position z_hi is ~4x farther from the origin than digit positions, making it hard to confuse with any digit.

Sinusoidal Digit Positions (Top View: dims 0-1)

The 10 digit positions trace a circle (frozen sinusoidal) with a slight upward drift in dim2.

Position Norms

The carry position (z_hi) has norm ~16.2, larger than digit positions (~3.5). This separation helps the model distinguish the carry slot from digits.

Attention Pattern

Q and K are computed from position only (not content), so the attention pattern is fixed regardless of the input numbers. A −29.4-degree phase rotation (qk_dim=4) shifts Q to create lookahead: each answer digit Z[i] attends to nearby X and Y positions, gathering digits for carry computation.

Carry Propagation

The most remarkable part: carries propagate through autoregressive generation. Each generated Z[i] token is fed back as input for generating Z[i+1]. The carry information is encoded in the token embedding of the previous answer digit.

Digit-by-Digit Carry Chain

Green = no carry out. Red = carry out. The carry from position i is communicated through the Z[i] token fed back as input.

How carry propagation works: The FFN has 2 hidden units: one fires when the digit sum + carry ≥ 10 (carry case), the other when it's < 10 (no carry). The Z[i] token that gets fed back as input encodes whether a carry occurred, because the token embedding's y-coordinate shifts the next column's sum up or down.

Output Logits

The final 2D projection is dot-producted with each digit's circular embedding to produce 10 logits. The softmax picks the answer digit.
